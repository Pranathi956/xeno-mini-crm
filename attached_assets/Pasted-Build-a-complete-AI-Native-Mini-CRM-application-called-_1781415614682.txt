Build a complete AI-Native Mini CRM application called "Xeno Mini CRM" with the following exact specifications:

## PROJECT STRUCTURE
Create two separate services:
1. /backend - FastAPI Python backend (main CRM)
2. /channel-service - FastAPI Python stub channel service
3. /frontend - React frontend

## DATABASE (SQLite, backend/crm.db)
Create these tables:
- customers: id, name, email, phone, city, total_spent, visit_count, last_purchase_date, created_at
- orders: id, customer_id, amount, product_name, status, created_at
- campaigns: id, name, segment_query, message, channel, status, created_at, sent_by_ai
- communications: id, campaign_id, customer_id, message, channel, status (pending/sent/delivered/failed/opened/clicked), created_at, updated_at
- campaign_stats: campaign_id, total_sent, delivered, failed, opened, clicked

## BACKEND (backend/main.py) - FastAPI on port 8000
Create these APIs:

### Data APIs
- POST /api/customers/bulk - insert list of customers
- POST /api/orders/bulk - insert list of orders
- GET /api/customers - list all customers with filters
- GET /api/stats/overview - total customers, orders, revenue

### Segment API
- POST /api/segments/preview - takes a SQL WHERE clause, returns matching customers count + sample

### Campaign APIs
- POST /api/campaigns - create campaign with {name, segment_query, message, channel, sent_by_ai}
- POST /api/campaigns/{id}/send - triggers sending: queries customers by segment, creates communication records, calls channel service for each
- GET /api/campaigns - list all campaigns
- GET /api/campaigns/{id}/stats - return campaign stats

### Receipt API (called by channel service)
- POST /api/receipts - receives {communication_id, status} and updates communication status and campaign_stats

### AI Chat API
- POST /api/ai/chat - takes {message, history} from marketer
- Use Groq API with model "llama3-70b-8192"
- Install groq python library: pip install groq
- Call Groq like this:
  from groq import Groq
  client = Groq(api_key=groq_api_key)
  response = client.chat.completions.create(
    model="llama3-70b-8192",
    messages=[{"role": "system", "content": SYSTEM_PROMPT}, ...history..., {"role": "user", "content": message}]
  )
- Read groq_api_key from request header X-API-Key
- Return response and optionally {action: "create_campaign", segment_query, message_draft, channel}

The AI system prompt should be:
"You are a CRM assistant for an Indian retail brand. You help marketers reach their customers. When a marketer describes a campaign goal, extract: 1) the customer segment as a SQL WHERE clause on the customers table (fields: total_spent, visit_count, last_purchase_date, city), 2) a personalized message draft in friendly tone, 3) recommended channel (whatsapp/sms/email). Always respond ONLY in JSON with no extra text: {reply: string, action: 'create_campaign' or null, segment_query: string or null, message_draft: string or null, channel: string or null}"

### Seed Data API
- POST /api/seed - insert 50 realistic fake Indian customers and 200 fake orders (names like Priya, Rahul, Ananya, Vikram etc., cities: Mumbai, Delhi, Bangalore, Chennai, Hyderabad, Pune, spending between 500-50000 INR, purchase dates spread over last 12 months)

## CHANNEL SERVICE (channel-service/main.py) - FastAPI on port 8001
- POST /send - receives {communication_id, customer_id, message, channel}
- Wait 2-5 seconds randomly (use asyncio.sleep)
- Then call backend POST /api/receipts with realistic outcomes using httpx async:
  - First callback: status="delivered" (80% of time) or status="failed" (20%)
  - If delivered: after 3 more seconds, 60% chance send status="opened"
  - If opened: after 2 more seconds, 40% chance send status="clicked"
- Use httpx AsyncClient to call back to http://localhost:8000/api/receipts

## CRITICAL IMPLEMENTATION RULES:
1. Use SQLAlchemy ORM for all DB operations, not raw SQL
2. Add CORS middleware to backend: allow_origins=["*"], allow_methods=["*"], allow_headers=["*"]
3. All FastAPI endpoints must be async
4. Frontend reads Groq API key from localStorage and sends it as header X-API-Key with every /api/ai/chat request
5. Backend /api/ai/chat reads key from request header X-API-Key — do NOT hardcode any API key
6. Campaign send must be non-blocking: use asyncio.create_task() to fire channel service calls, return immediately
7. Channel service URL in backend should be configurable via env variable CHANNEL_SERVICE_URL defaulting to http://localhost:8001
8. Parse AI response safely — strip any markdown/json fences before JSON.parse
9. All amounts in Indian Rupees (₹)
10. Make the UI look professional and modern — this is a portfolio/interview project
